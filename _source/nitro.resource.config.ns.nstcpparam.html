ackonpush¶

Send immediate positive acknowledgement (ACK) on receipt of TCP packets with PUSH flag.<br/>Default value: ENABLED<br/>Possible values = ENABLED, DISABLED.

builtin¶

Flag to determine if the tcp param is built-in or not.<br/>Possible values = MODIFIABLE, DELETABLE, IMMUTABLE, PARTITION_ALL.

connflushifnomem¶

Flush an existing connection if no memory can be obtained for new connection. HALF_CLOSED_AND_IDLE: Flush a connection that is closed by us but not by peer, or failing that, a connection that is past configured idle time. New connection fails if no such connection can be found. FIFO: If no half-closed or idle connection can be found, flush the oldest non-management connection, even if it is active. New connection fails if the oldest few connections are management connections. Note: If you enable this setting, you should also consider lowering the zombie timeout and half-close timeout, while setting the NetScaler timeout. See Also: connFlushThres argument below. <br/>Default value: 5<br/>Possible values = NONE, HALFCLOSED_AND_IDLE, FIFO.

connflushthres¶

Flush an existing connection (as configured through -connFlushIfNoMem FIFO) if the system has more than specified number of connections, and a new connection is to be established. Note: This value may be rounded down to be a whole multiple of the number of packet engines running.<br/>Minimum length = 1.

delayedack¶

Timeout for TCP delayed ACK, in milliseconds.<br/>Default value: 100<br/>Minimum length = 10<br/>Maximum length = 300.

downstaterst¶

Flag to switch on RST on down services.<br/>Default value: DISABLED<br/>Possible values = ENABLED, DISABLED.
