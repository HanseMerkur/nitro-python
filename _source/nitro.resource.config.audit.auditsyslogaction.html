ip¶

The resolved IP address of the syslog server.

lbvservername¶

Name of the LB vserver. Mutually exclusive with syslog serverIP/serverName.<br/>Minimum length = 1<br/>Maximum length = 127.

logfacility¶

Facility value, as defined in RFC 3164, assigned to the log message. Log facility values are numbers 0 to 7 (LOCAL0 through LOCAL7). Each number indicates where a specific message originated from, such as the NetScaler appliance itself, the VPN, or external.<br/>Possible values = LOCAL0, LOCAL1, LOCAL2, LOCAL3, LOCAL4, LOCAL5, LOCAL6, LOCAL7.

loglevel¶

Audit log level, which specifies the types of events to log. Available values function as follows: * ALL - All events. * EMERGENCY - Events that indicate an immediate crisis on the server. * ALERT - Events that might require action. * CRITICAL - Events that indicate an imminent server crisis. * ERROR - Events that indicate some type of error. * WARNING - Events that require action in the near future. * NOTICE - Events that the administrator should know about. * INFORMATIONAL - All but low-level events. * DEBUG - All events, in extreme detail. * NONE - No events.<br/>Possible values = ALL, EMERGENCY, ALERT, CRITICAL, ERROR, WARNING, NOTICE, INFORMATIONAL, DEBUG, NONE.

lsn¶

Log lsn info.<br/>Possible values = ENABLED, DISABLED.

maxlogdatasizetohold¶

Max size of log data that can be held in NSB chain of server info.<br/>Default value: 500<br/>Minimum length = 50<br/>Maximum length = 25600.

name¶

Name of the syslog action. Must begin with a letter, number, or the underscore character (_), and must contain only letters, numbers, and the hyphen (-), period (.) pound (#), space ( ), at (@), equals (=), colon (:), and underscore characters. Cannot be changed after the syslog action is added. The following requirement applies only to the NetScaler CLI: If the name includes one or more spaces, enclose the name in double or single quotation marks (for example, “my syslog action” or ‘my syslog action).<br/>Minimum length = 1.

netprofile¶

Name of the network profile. The SNIP configured in the network profile will be used as source IP while sending log messages.<br/>Minimum length = 1<br/>Maximum length = 127.

serverdomainname¶

SYSLOG server name as a FQDN. Mutually exclusive with serverIP/lbVserverName.<br/>Minimum length = 1<br/>Maximum length = 255.

serverip¶

IP address of the syslog server.<br/>Minimum length = 1.

serverport¶

Port on which the syslog server accepts connections.<br/>Minimum length = 1.

subscriberlog¶

Log subscriber session event information.<br/>Possible values = ENABLED, DISABLED.

tcp¶

Log TCP messages.<br/>Possible values = NONE, ALL.

tcpprofilename¶

Name of the TCP profile whose settings are to be applied to the audit server info to tune the TCP connection parameters.<br/>Minimum length = 1<br/>Maximum length = 127.

timezone¶

Time zone used for date and timestamps in the logs. Supported settings are: * GMT_TIME. Coordinated Universal time. * LOCAL_TIME. Use the server’s timezone setting.<br/>Possible values = GMT_TIME, LOCAL_TIME.

transport¶

Transport type used to send auditlogs to syslog server. Default type is UDP.<br/>Possible values = TCP, UDP.
