crlname¶

Name for the Certificate Revocation List (CRL). Must begin with an ASCII alphanumeric or underscore (_) character, and must contain only ASCII alphanumeric, underscore, hash (#), period (.), space, colon (:), at (@), equals (=), and hyphen (-) characters. Cannot be changed after the CRL is created. The following requirement applies only to the NetScaler CLI: If the name includes one or more spaces, enclose the name in double or single quotation marks (for example, “my crl” or ‘my crl’).<br/>Minimum length = 1.

crlpath¶

Path to the CRL file. /var/netscaler/ssl/ is the default path.<br/>Minimum length = 1.

day¶

Day on which to refresh the CRL, or, if the Interval parameter is not set, the number of days after which to refresh the CRL. If Interval is set to MONTHLY, specify the date. If Interval is set to WEEKLY, specify the day of the week (for example, Sun=0 and Sat=6). This parameter is not applicable if the Interval is set to DAILY.<br/>Maximum length = 0x1F.

daystoexpiration¶

Number of days remaining for the CRL to expire.

indexfile¶

Name of and, optionally, path to the file containing the serial numbers of all the certificates that are revoked. Revoked certificates are appended to the file. /nsconfig/ssl/ is the default path.<br/>Maximum length = 63.

inform¶

Input format of the CRL file. The two formats supported on the appliance are: PEM - Privacy Enhanced Mail. DER - Distinguished Encoding Rule.<br/>Default value: PEM<br/>Possible values = DER, PEM.

interval¶

CRL refresh interval. Use the NONE setting to unset this parameter.<br/>Possible values = MONTHLY, WEEKLY, DAILY, NONE.

issuer¶

Issuer name.

lastupdate¶

Last update time.

lastupdatetime¶

Last CRL refresh time.

method¶

Method for CRL refresh. If LDAP is selected, specify the method, CA certificate, base DN, port, and LDAP server name. If HTTP is selected, specify the CA certificate, method, URL, and port. Cannot be changed after a CRL is added.<br/>Possible values = HTTP, LDAP.

nextupdate¶

Next update time.

password¶

Password to access the CRL in the LDAP repository if access to the LDAP repository is restricted or anonymous access is not allowed.<br/>Minimum length = 1.

port¶

Port for the LDAP server.<br/>Minimum length = 1.

refresh¶

Set CRL auto refresh.<br/>Possible values = ENABLED, DISABLED.

revoke¶

Name of and, optionally, path to the certificate to be revoked. /nsconfig/ssl/ is the default path.<br/>Maximum length = 63.

scope¶

Extent of the search operation on the LDAP server. Available settings function as follows: One - One level below Base DN. Base - Exactly the same level as Base DN.<br/>Default value: One<br/>Possible values = Base, One.

server¶

IP address of the LDAP server from which to fetch the CRLs.<br/>Minimum length = 1.

signaturealgo¶

Signature algorithm.

time¶

Time, in hours (1-24) and minutes (1-60), at which to refresh the CRL.
